.Net Framework Last updated: 01-08-2023

The .NET Framework is a proprietary software framework developed by Microsoft that runs primarily on Microsoft Windows. It was the predominant implementation of the Common Language Infrastructure until being superseded by the cross-platform .NET project.

What is C#

C# is a programming language developed by Microsoft as part of the .NET framework. It is a modern, object-oriented language that is designed to be simple and easy to learn, yet powerful enough to build a wide range of applications.

Features of C#:

.NET Framework Structure

NET Framework is a software development framework for building and running applications on Windows.

Common Language Runtime (CLR)

The CLR is the runtime environment for .NET applications. It is responsible for managing the execution of .NET code, including memory management, security, and exception handling.

Base Class Library (BCL)

The BCL is a library of classes and methods that provides a wide range of functionality for .NET applications. This includes classes for common tasks such as reading and writing files, connecting to databases, and working with the web.

Architecture of .NET Framework

The .NET framework has a modular architecture, which means that it is composed of a number of smaller components that can be used independently or combined together. These components include the CLR, the BCL, and other libraries and frameworks such as ASP.NET and Windows Forms.

Assembly Use

An assembly is a file that is automatically generated by the compiler upon successful compilation of every . NET application. It can be either a Dynamic Link Library or an executable file.

What is an Assembly

An assembly is a compiled code library for .NET applications. It can contain code written in any .NET language, such as C# or VB

Types of Assemblies

There are two main types of assemblies in the .NET framework: private assemblies and shared assemblies. Private assemblies are used by a single application, and are stored in the application's installation directory. Shared assemblies, on the other hand, are intended to be used by multiple applications, and are stored in a central location called the global assembly cache (GAC).

Creating and Using assemblies

To create an assembly, you can use a .NET compiler, such as csc.exe for C# or vbc.exe for VB.NET. Once you have created an assembly, you can use it in your .NET application by adding a reference to it in your project.
